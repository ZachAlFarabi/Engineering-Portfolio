● Objectives

The main basis for the program is based on a text RPG (role playing game), or rogue-like adventure game. As it is a rogue-like adventure game, it works on having 1 game per run. The program exists in 2 separate systems of gameplay: Shopping, Battling, as well as an extension inventory. As such, it relies on much of dynamic memory and other self-initiated learning concepts.

Go back!

● Technical Details

  • Implemented core game systems (shop, battle, inventory) using nested loops, conditional logic, and modular functions to handle dynamic gameplay within a single run.
  • Used global variables and dynamic memory management so that player stats, items, and state information persist across loops and function calls.
  • Created custom graphics with MATLAB figure, line, set, and color functions to display character sprites and a dynamically updating health bar.
  • Built the program incrementally through 8 documented versions, with detailed comments and version summaries, and tested each system using a mix of black-box and white-box methods.
